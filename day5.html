Day 5: API Authentication

Basic Authentication

Basic Authentication is a simple authentication scheme built into the HTTP protocol. It sends credentials in the Authorization header.

How it works:

  1. Username and password are combined with a colon
  2. The resulting string is encoded in Base64
  3. The encoded string is sent in the Authorization header

Example Request:

GET https://api.github.com/user
Authorization: Basic base64(username:password)

Bearer Token Authentication

Bearer Token authentication is a common method where a token is sent in the Authorization header.

Key Characteristics:

  • Token is obtained through OAuth or similar process
  • Token is sent in the Authorization header
  • More secure than Basic Auth
  • Can be revoked without changing passwords

Example Request:

GET https://api.github.com/user
Authorization: Bearer your_token_here

OAuth 2.0

OAuth 2.0 is an authorization framework that enables applications to obtain limited access to user accounts.

Common OAuth Flows:

  • Authorization Code Flow
  • Implicit Flow
  • Client Credentials Flow
  • Resource Owner Password Credentials Flow

Example GitHub OAuth Flow:

  1. Register your application with GitHub
  2. Get client ID and client secret
  3. Redirect users to GitHub's authorization page
  4. Exchange code for access token
  5. Use access token for API requests
