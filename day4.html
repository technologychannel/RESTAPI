Day 4: PUT & DELETE Requests

PUT Requests

PUT requests are used to update existing resources. They are idempotent, meaning multiple identical requests should have the same effect as a single request.

Key Characteristics:

  • Updates the entire resource
  • Requires the complete resource representation
  • Idempotent (safe to retry)
  • Returns 200 OK or 204 No Content on success

Example PUT Request:

PUT https://jsonplaceholder.typicode.com/posts/1
Content-Type: application/json

{
    "id": 1,
    "title": "Updated Post",
    "body": "This post has been updated",
    "userId": 1
}

DELETE Requests

DELETE requests are used to remove resources from the server. Like PUT, they are also idempotent.

Key Characteristics:

  • Removes the specified resource
  • Idempotent (safe to retry)
  • Returns 204 No Content on success
  • May return 404 if resource doesn't exist

Example DELETE Request:

DELETE https://jsonplaceholder.typicode.com/posts/1

PUT vs PATCH

Feature PUT PATCH
Purpose Update entire resource Update partial resource
Data Required Complete resource Only changed fields
Idempotent Yes Not necessarily
