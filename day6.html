Day 6: Headers & Query Parameters

HTTP Headers

HTTP headers are key-value pairs that provide additional information about the request or response.

Common Request Headers:

Header Description Example
Content-Type Specifies the media type of the resource application/json
Accept Indicates what media types the client can understand application/json
Authorization Contains credentials for authentication Bearer token

Query Parameters

Query parameters are key-value pairs that appear after the question mark (?) in a URL. They are used to filter, sort, or paginate data.

Common Use Cases:

  • Filtering results
  • Sorting data
  • Pagination
  • Searching

Example URL with Query Parameters:

https://api.example.com/users?page=1&limit=10&sort=name&filter=active

Practical Examples

Example 1: Filtering with Query Parameters

GET https://jsonplaceholder.typicode.com/posts?userId=1

This request will return all posts from user with ID 1.

Example 2: Pagination

GET https://api.example.com/users?page=2&limit=20

This request will return the second page of users, with 20 users per page.

Example 3: Sorting

GET https://api.example.com/products?sort=price&order=desc

This request will return products sorted by price in descending order.
