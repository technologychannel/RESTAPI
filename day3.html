Day 3: HTTP Status Codes

Understanding HTTP Status Codes

HTTP status codes are three-digit numbers that indicate the outcome of an HTTP request. They are grouped into five classes:

2xx - Success

  • 200 OK - Request succeeded
  • 201 Created - Resource created
  • 204 No Content - Success but no content

3xx - Redirection

  • 301 Moved Permanently
  • 302 Found (Temporary Redirect)
  • 304 Not Modified

4xx - Client Error

  • 400 Bad Request
  • 401 Unauthorized
  • 403 Forbidden
  • 404 Not Found

5xx - Server Error

  • 500 Internal Server Error
  • 502 Bad Gateway
  • 503 Service Unavailable

Real-Life Examples

Success Scenario (200 OK)

GET https://api.example.com/users/1
Response: 200 OK
{
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com"
}

Resource Not Found (404)

GET https://api.example.com/users/999
Response: 404 Not Found
{
    "error": "User not found",
    "message": "The requested user does not exist"
}

Authentication Error (401)

GET https://api.example.com/protected-resource
Response: 401 Unauthorized
{
    "error": "Unauthorized",
    "message": "Authentication required"
}

Best Practices

  • Always use appropriate status codes
  • Include meaningful error messages
  • Use consistent error response format
  • Document all possible status codes in API documentation
  • Handle errors gracefully on the client side
