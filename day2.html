Day 2: GET/POST Requests

GET Requests

GET requests are used to retrieve data from a server. They are the most common type of HTTP request and are used when you want to fetch information.

Key Characteristics:

  • Data is sent in the URL as query parameters
  • No request body
  • Can be cached
  • Should be idempotent (same request = same result)

Example GET Request:

GET https://jsonplaceholder.typicode.com/posts/1
Accept: application/json

POST Requests

POST requests are used to send data to a server to create a new resource. They are commonly used when submitting forms or creating new records.

Key Characteristics:

  • Data is sent in the request body
  • Not idempotent (same request can create multiple resources)
  • Cannot be cached
  • Should return 201 Created on success

Example POST Request:

POST https://jsonplaceholder.typicode.com/posts
Content-Type: application/json

{
    "title": "New Post",
    "body": "This is a new post",
    "userId": 1
}

Exercise 1: GET Request

  2. Create a new GET request
  3. Set URL to: https://jsonplaceholder.typicode.com/posts
  4. Add header: Accept: application/json
  5. Send request and observe the response

Exercise 2: POST Request

  2. Set URL to: https://jsonplaceholder.typicode.com/posts
  3. Add header: Content-Type: application/json
  4. Add request body:

    "title": "My First Post",
    "body": "This is my first post using Insomnia",
    "userId": 1
}
  5. Send request and observe the response
