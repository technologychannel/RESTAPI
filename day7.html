Day 7: Mock API Design

What is a Mock API?

A mock API is a simulated version of a real API that mimics its behavior. It's used for testing and development when the actual API is not available or not ready.

Benefits of Mock APIs:

  • Parallel development of frontend and backend
  • Testing without affecting production data
  • Simulating different scenarios and responses
  • Reducing dependencies during development

Popular Mock API Tools

JSON Server

A simple tool to create a REST API with zero coding.

npm install -g json-server
json-server --watch db.json

Mockoon

A desktop application to create mock APIs with a user-friendly interface.

Creating a Mock API with JSON Server

Step 1: Create db.json

{
  "users": [
    {
      "id": 1,
      "name": "John Doe",
      "email": "john@example.com"
    },
    {
      "id": 2,
      "name": "Jane Smith",
      "email": "jane@example.com"
    }
  ],
  "posts": [
    {
      "id": 1,
      "title": "First Post",
      "body": "This is the first post",
      "userId": 1
    }
  ]
}

Step 2: Start the Server

json-server --watch db.json --port 3000

Available Endpoints:

  • GET /users - Get all users
  • GET /users/1 - Get user with id 1
  • POST /users - Create new user
  • PUT /users/1 - Update user
  • DELETE /users/1 - Delete user
